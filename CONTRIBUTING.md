# Contributing to Terragrunt-live

Thank you for considering contributing! This project is intended to be a
community-friendly template for Terragrunt-based Azure deployments.

## Getting Started

1. Fork the repository and clone your fork.
2. Create a new branch for your feature or bug fix:
   ```bash
   git checkout -b feature/my-new-feature
   ```
3. Make your changes, keeping them focused and well-tested.
4. Add or update documentation as needed in `README.md` or other relevant
   files.
5. Commit your changes with a descriptive message.
6. Push the branch and open a pull request against the `main` branch.

## Code Style

- Keep Terragrunt/Terraform code formatted (`terraform fmt`).
- YAML files should be valid and use 2-space indentation for readability.
- Document any non-obvious behavior or configuration options.

## Issues & Feature Requests

Please open an issue in the GitHub repository describing the problem or
suggested enhancement. When filing an issue, include:

- A clear title and description
- Steps to reproduce (for bugs)
- Expected versus actual behavior
- Any relevant configuration snippets (redact sensitive data)
